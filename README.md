## 변수,상수 작성 과제

1. 수정될 수 있는 값 let과 변하지 않거나 변해서는 안되는 상수 const의 구분 사용
2. 적절한 변수 작명
3. camel case, snake case 등 일관성있는 코드 컨벤션

능력치를 요구하는 과제라고 생각하였습니다.

상수의 경우 문제에서 상수를 사용하여 선언하라고 명시되어 있었기에 const를 사용하였습니다.

그 외에는 let을 사용하여 선언하였습니다.

8번 문제의 경우 각 배송 상태명은 변경되지 않기에 const를 사용하였고, 현재 배송상태는 언제든지 변경될 수 있기 때문에 let을 사용하여 선언하였습니다.

## 함수 작성 과제

1. 각 문제마다 함수 선언, 함수 표현식, 화살표 함수 등 다양한 함수 타입 사용 요구
2. 각 함수의 매개변수와 반환값 타입이 정해져있음
3. 조건문을 배우지 않았기 때문에 ‘객체’와 ‘연산자’를 사용하여 각 상태와 상황에 맞는 결과값 계산

Javascript를 미리 공부했던 입장에서 조건문 없이 구현해야하는 것이 무엇보다 가장 당황스러웠습니다. 처음에는 삼항 연산자를 사용하여 계산하였으나 진도 범위 밖이었기 때문에 많은 고민 끝에 객체를 이용하여 함수를 구현하였습니다. 특히나 객체 대괄호를 활용하면 매개변수로 받는 값을 통해서 바로 객체의 값을 불러올 수 있기 때문에 문제를 해결하기 적합했습니다.

return문에서 멋들어지게 구현하는 방법도 있었으나 조건문 없이 구현하는 것이 처음이거나 처음 프로그래밍을 배웠을 오래 전의 일이기 때문에 실수하지 않기 위해서 많은 변수를 만들어 콘솔을 찍어가며 테스트 하였습니다.
